Use the reference sheet only as the visual/filter template. Do not copy its stale rows as final data unless rebuilding a reference-only workbook.

Output sheet:
- Sheet name: BEV Series Name Table
- Real table range: A1:D{last_row}
- Headers in row 1: Brand | รุ่นรถ | รุ่นรถ2 | Powertrain
- No merged cells
- No Excel table object required
- Add AutoFilter over A1:D{last_row}

Recommended data source:
1. Start from the cleaned/model-side rows, not fuel-only raw1.
2. Source Brand from ยี่ห้อรถ2.
3. Source รุ่นรถ from raw2/model data column รุ่นรถ.
4. Source รุ่นรถ2 from refer/model2_map.csv, keyed by upper/trimmed รุ่นรถ. If missing, keep รุ่นรถ as fallback.
5. Source Powertrain in this priority:
   - If raw/model row has powertrain_raw, map it through master powertrain.
   - Else use the reference BEV Series Name Table mapping by รุ่นรถ.
   - Else use OTH.

Recommended rows to include:
- One unique row per Brand + รุ่นรถ + รุ่นรถ2 + Powertrain.
- Sort by Brand, then รุ่นรถ2, then รุ่นรถ.
- Keep Powertrain categories: BEV Major, BEV, OTH.

Recommended default filter/hide behavior:
- Add AutoFilter to A1:D{last_row}.
- Match the reference workbook's visual default by hiding OTH rows.
- The reference also hides row 2 (AION ES, BEV Major), but that looks manual/stale. Do not reproduce that special hide unless the user explicitly wants exact reference quirks.
- If you need exact Excel filter metadata, filter field is column D / colId=3.
- Best xlsxwriter equivalent:
    ws.autofilter(0, 0, last_row - 1, 3)
    for each data row where Powertrain == "OTH": ws.set_row(row_index, None, None, {"hidden": True})
