The product exists to compute two facts: "what powertrain / brand / month is this row" and "sum units by group over a period". Each is re-implemented 3–5 times across Python and TypeScript, from different source columns, with no shared module. Every candidate below is a symptom of that. The seam between stages is two parquet files whose schema is an unwritten contract of Thai column-name string literals.

Tackle first

1 · Unify the classification module

It's the fact the whole product exists to compute, and it fails the deletion test today — three classifiers that merely happen to mostly agree. It already shows present drift (the Deepal+ChangAn / Deepal + Changan mismatch) and a latent one (a new fuel type never reaches the dashboard). Deepening it concentrates classification in one module and turns the parquet column into a real seam — which is the precondition for #3 (its cleanest extraction) and removes the frontend re-aliasing in #5.
