  ---
  Data Flow Summary

  raw data (File 1 + File 2)
          ↓
     build_cleaned.py
          ↓
   master Model.xlsx (cleaned Data sheet)
   test_model_cleaned.parquet ──────────────────┐
          ↓                                     ↓
    build_BEV.py                         build_analyst.py
          ↓                                     ↓
   master Model.xlsx                   ...(analyst).xlsx
   (+ BEV sheets added)

  The key design: each step writes a file the next step reads, so they can run independently. The .parquet file is the
  handoff between Step 1 → Steps 2 and 3.
